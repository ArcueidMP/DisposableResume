# Security

DisposableResume is currently pre-release.

## Reporting

Please avoid sharing real resumes, names, email addresses, phone numbers,
addresses, employers, or other personal data in reports.

For sensitive issues, use private vulnerability reporting if it is enabled for
the repository. If private reporting is unavailable, open a minimal public issue
that describes the category of the problem without including personal data or
resume contents.

Privacy-sensitive issues include:

- Resume data sent over the network.
- Resume data persisted unexpectedly.
- Resume data written to logs.
- Resume data appearing in URLs.
- PDF export moving to backend rendering.
- Analytics, telemetry, tracking pixels, or session replay added to the app.

## Supported Versions

No stable release has been published yet. Security expectations apply to the
current default branch and active feature branches.
